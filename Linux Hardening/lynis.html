Lynis Security Audit

Host-based security scanner — inspect, score, harden

Debian / Ubuntu Open Source Agentless

What does Lynis actually do?

Lynis is a host-based security auditing tool. When you run it, it performs over 200 individual checks across your system — examining kernel parameters, filesystem permissions, running services, user accounts, network configuration, installed software, and authentication settings — and cross-references its findings against security benchmarks.

It doesn't change anything on your system. It only reads and reports. The output is a prioritised list of weaknesses and hardening opportunities, each with an explanation and a suggested fix. The goal isn't a perfect score — it's a clear, actionable picture of where your system stands.

STEP 01

Verify Installation

Lynis installs its binary to /usr/sbin/lynis — a directory reserved for system administration tools. Rather than using which (which only searches your current PATH and may miss /usr/sbin), check the exact known path directly:

terminal
ls /usr/sbin/lynis

If the path prints back, Lynis is installed — skip to Step 03. "No such file or directory" means proceed to Step 02.

STEP 02

Install Lynis

Only run this if the check above confirmed Lynis is missing. The package is available in the standard Debian/Ubuntu repositories:

terminal
sudo apt update && sudo apt install lynis -y

Note that the repository version may lag slightly behind the upstream release. For the latest version, you can also install directly from the CISOfy repository, though the apt version is sufficient for most purposes.

STEP 03

Run the Full System Audit

The audit system command runs all available tests against the local host. Running as root with sudo is required — many checks need access to protected files and kernel parameters that are not readable by normal users. Without it, Lynis will skip large sections of the audit and your score will be artificially low:

terminal
sudo lynis audit system

The scan typically takes 1–3 minutes. You'll see each test category scroll by as it runs. Categories covered include:

Boot & Services
Bootloader config, running services, init system
Kernel
Kernel version, sysctl parameters, module loading
Authentication
PAM config, password policies, sudo rules, SSH settings
Filesystem
Mount options, world-writable files, SUID/SGID binaries
Networking
Open ports, firewall state, network interface config
Software
Installed packages, compiler tools, malware scanners
STEP 04

Reading the Results

Scroll to the bottom of the terminal output. The three most important sections are:

WARNINGS
Active issues that should be addressed. These are configurations or conditions that are directly insecure — for example, a world-readable private key file or SSH allowing root login with a password.
SUGGESTIONS
Hardening opportunities. These are settings that aren't necessarily broken but could be tightened — for example, disabling unused kernel modules or enabling automatic security updates.
HARDENING INDEX
A score from 0 to 100 reflecting the proportion of checks your system passed. It's a relative measure, not a pass/fail — a freshly installed system typically scores in the 55–65 range. Aim for 75+ as a reasonable hardened baseline.
0Typical fresh install: ~60Target: 75+100

Higher isn't always better — some suggestions involve trade-offs between security and usability. Read each one before applying it.

The full audit log is written to /var/log/lynis.log and a report file to /var/log/lynis-report.dat. Both persist between runs and are useful for tracking changes over time.
