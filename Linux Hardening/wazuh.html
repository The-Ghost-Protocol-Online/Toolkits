Wazuh Setup Guide

Open-source SIEM, intrusion detection, and compliance monitoring

Debian / Ubuntu All-in-One Install v4.x

What is Wazuh?

Wazuh is an open-source security platform that combines a SIEM (Security Information and Event Management) with XDR (Extended Detection and Response) capabilities. In practical terms, it collects and analyses logs from across your system, detects intrusions and policy violations, monitors file integrity, and generates compliance reports — all from a single platform.

The all-in-one installation deploys three components onto a single machine:

Wazuh Manager
The core engine. Receives data from agents, processes alerts, and enforces rules.
Wazuh Indexer
An OpenSearch instance that stores and indexes all alert data for querying.
Wazuh Dashboard
A web UI (OpenSearch Dashboards) for viewing alerts, compliance reports, and agent status.
STEP 01

Verify System Requirements

Wazuh is considerably more resource-intensive than a typical service because it runs a full search engine (the Indexer) alongside the Manager. The Indexer alone requires a substantial JVM heap. Check your machine meets the minimum specs before continuing:

terminal
# Available RAM — the Indexer's JVM needs headroom to breathe free -h # CPU core count — parallel indexing is CPU-bound nproc
Minimum RAM
4 GB
The Indexer (OpenSearch) alone allocates ~2 GB of JVM heap by default. Below 4 GB the installation will likely fail or the service will OOM-kill shortly after starting.
Minimum CPU Cores
2 Cores
OpenSearch is heavily threaded. A single core will cause severe performance degradation during indexing and search.
STEP 02

Download the Installer

Wazuh provides an official shell script that handles the full installation. Use curl with the -sL flags: -s suppresses the progress meter (keeps output clean) and -L follows redirects automatically. The -o flag saves it to a named file rather than printing to stdout:

terminal
curl -sL https://packages.wazuh.com/4.x/wazuh-install.sh -o wazuh-install.sh

Before running any downloaded script, it's good practice to confirm you actually received a shell script and not an error page. If the download was intercepted or the URL returned an error, you may get an XML document instead. A quick check:

terminal
head -1 wazuh-install.sh

The first line should read #!/bin/bash. If it starts with <?xml instead, the download failed — delete the file and re-run the curl command above. Running an XML document as a shell script produces the "unexpected token" error you may have seen.

STEP 03

Run the All-in-One Installer

The -a flag tells the script to perform an all-in-one deployment — installing and configuring the Manager, Indexer, and Dashboard on this single machine. This is the correct mode for a self-contained installation. Expect it to take 5–10 minutes as it adds the Wazuh APT repository, installs all three components, generates TLS certificates, and starts the services:

terminal
sudo bash wazuh-install.sh -a

Do not interrupt the script once it starts. If it fails partway through, run it again with the -o flag to overwrite the partial installation: sudo bash wazuh-install.sh -a -o

STEP 04

Verify the Installation

Once the script finishes, check that the core service is running. The Manager is the heart of Wazuh — if this is healthy, the installation succeeded:

terminal
sudo systemctl status wazuh-manager
Look for Active: active (running) in the output. If it shows failed, check journalctl -u wazuh-manager -n 50 for the error details.

The installer prints the dashboard URL and auto-generated admin credentials at the end of its output. The dashboard is accessible at https://<your-server-ip> on port 443. Save those credentials — they are only shown once.
